# iPhone Bridge

把 iPhone 当作可以主动请求的现实端能力。手机端具体执行什么，取决于用户已经配置并测试成功的苹果快捷指令。

## 已注册操作

默认没有已注册操作。只有用户已经在自己的 iPhone 上配置并完成真实测试的操作，才能添加到这里。

例如 `闹钟`、`查找位置`、`截屏` 都只是可能的操作名称，不代表默认可用。

## 调用已注册操作

确认操作已经列在“已注册操作”后，在项目根目录运行：

```powershell
python "scripts/abilities/connect_iphone/send_to_iphone.py" "主题" "内容"
```

主题必须与注册名称完全一致，主题和内容分别放在引号中。命令输出 `已发送` 只表示请求邮件已经交给邮箱服务器，不代表手机已经完成操作。

发送后等待手机反馈，不要连续重复发送，也不要自行启动 `receive_from_iphone.py`。后台监听器会把手机返回的文字或图片送进当前会话。

收到图片路径后，使用已有识图能力查看图片，再结合当前对话正常回应。

## 所需操作尚未注册

发现需要的操作不在注册列表时：

1. 明确说明目前还没有配置这项手机能力，不要发送未知主题。
2. 提出可以一起添加，并说明大致能否通过苹果快捷指令实现。
3. 用户同意后，指导其配置手机端快捷指令；不要声称尚未测试的能力已经可用。
4. 配置完成后进行一次真实测试。
5. 只有用户确认测试成功并要求登记时，才把新主题、用途和正文格式加入本文件的“已注册操作”。

给操作命名时使用简短、唯一、不会与普通反馈混淆的邮件主题。根据操作需要定义正文参数；没有参数时正文留空。

## 指导新增苹果快捷指令

请求方向：

```text
Agent 运行 send_to_iphone.py
→ 邮箱收到指定主题的请求邮件
→ iPhone 自动化按发件人和主题触发
→ 快捷指令执行手机操作
```

需要返回结果时：

```text
iPhone 快捷指令取得文字、位置或图片
→ 发送一封约定主题的反馈邮件
→ 文字放在邮件正文，图片作为邮件附件
→ 电脑监听器把正文或本地图片路径送给 Agent
```

指导用户时应覆盖：

- 在 iPhone“快捷指令”的“自动化”中，以收到指定发件人和指定主题的邮件作为触发条件。
- 关闭运行前询问或选择立即运行；具体名称以用户当前 iOS 显示为准。
- 添加真正需要执行的手机动作，并在需要时把请求邮件正文作为参数。
- 需要反馈时，用“发送电子邮件”把结果发回监听邮箱，主题与 `feedback_config.json` 中的路由一致。
- 返回图片时，把图片变量作为真实邮件附件发送，不要转成 Base64 或粘贴进正文。
- 关闭发送前显示编辑界面，否则自动化可能停在确认页面。
- 苹果系统可能限制某些动作在锁屏或后台运行；遇到限制时如实说明并通过实际测试确认。

## 边界

- 不读取或展示 `feedback_config.json` 中的邮箱授权码、Webhook Token 等凭证。
- 不修改注册列表，除非用户明确确认对应快捷指令已经测试成功并要求登记。
- 不把手机没有返回的信息编造为现实状态。
- 一封请求尚在等待反馈时，不重复发起相同请求。
