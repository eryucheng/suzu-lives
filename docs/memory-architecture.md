# 记忆架构

## 目标

持续保留 Claude Code 的会话 JSONL，不等于每轮都把整份文件放进模型窗口。记忆模块把“永久保存的数据”和“本轮需要看到的上下文”分开：主 JSONL 与历史库可以继续增长，模型每轮稳定看到的是第一人称摘要、近期完整原文，以及当前话题真正需要的一小段历史依据。

## 三类上下文

### 第一人称中长期摘要

定向压缩器启动独立的一次性 LLM，把既有摘要与本批即将退出短期范围的对话合并。摘要中的“我”始终是 Agent 自己，保留重要经历、关系变化、承诺、边界、计划、原因和未完成状态。

LLM 同时返回少量事件候选，但 JSONL 结构、UUID、父链、compact 边界、来源校验、备份与写入全部由确定性代码处理。

### 近期完整原文

默认至少保留最近 24 小时原文。24 小时内如果 live context 已超过 15k tokens，则提前建立新 checkpoint，并保留末尾约 5k tokens。切点落在完整用户消息边界，摘要 LLM 只读查看切点后的少量衔接内容，避免把进行中的事情错写成已经完成。

### 按需历史依据

退出短期范围的双方可见原文进入 `history.jsonl`，值得独立检索的经历进入 `events.jsonl`。RAG 根据当前消息选择：

- 一张事件卡片；
- 一条历史原话；
- 事件缺失时最多 3 条相关原文证据；
- 纯日期问题的一份当日／日期范围事件汇总；
- 没有可靠命中时不注入。

默认只给主 Agent 一个片段。RAG 负责提供证据，不负责解释证据或替 Agent 写最终答案。

## 一次压缩的顺序

```text
读取主 JSONL 并重建最新逻辑链
  → 依据 24 小时／15k token 规则选择切点
  → 标准化即将退出短期范围的双方原文
  → 一次性 LLM 更新第一人称摘要并提出事件候选
  → 备份并追加 compact 边界与摘要
  → 原文去重写入 history.jsonl
  → 事件校验后写入 events.jsonl
  → 可选：补齐 embeddings.jsonl
```

compact 通过追加新逻辑边界实现，不清理旧 JSONL。主 JSONL 或历史库在执行期间变化时，代码会拒绝覆盖或回滚；事件与向量索引属于可恢复的派生层，失败不会破坏已经完成的主记忆。

## 检索路线

`history.jsonl` 和 `events.jsonl` 共同构成可检索事实源。BM25 负责明确词语匹配，可选 Embedding 提供语义匹配，二者经过阈值与混合排序后才允许注入。

事件卡片的 `event_date` 表示事件实际发生或计划发生的日期，来源消息时间单独保存。相对日期问题先按 `Asia/Shanghai` 将“上周日”等表达解析成明确日期，再限定事件范围；没有对应事件时返回空，不用其他日期的内容填充。

## 数据边界

历史原文层排除：system、thinking、工具调用、工具结果、compact 摘要、`NO_REPLY`、运行时当前时间注入和已知自动化触发回合。过滤只排除运行噪声，不对正常关系对话做关键词审查。

事件层必须引用真实归档消息，无法可靠确定事件日期时保留未知，不允许用消息时间或压缩时间假装事件发生时间。

## 框架边界

compact 写入器针对 Claude Code JSONL 结构实现；Hook 针对 Claude Code `UserPromptSubmit`。标准历史格式、事件格式和检索选择原则相对独立。适配其他 Agent 时，只需替换“读取原始会话”“建立逻辑 checkpoint”和“向当前提示注入回忆”的接口。
